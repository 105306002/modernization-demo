## 🎯 核心功能

### API 端點

#### 1️⃣ 查詢帳戶餘額
- **端點：** `GET /api/account/{accountId}/balance`
- **功能：** 查詢指定帳號的當前餘額
- **回應格式：**
```json
{
  "accountId": "ACC001",
  "balance": 50000.00,
  "currency": "TWD"
}
```

#### 2️⃣ 查詢交易記錄
- **端點：** `GET /api/account/{accountId}/transactions`
- **功能：** 查詢指定帳號的近期交易記錄（模擬 3 筆）
- **回應格式：**
```json
[
  {
    "date": "2024-03-23T...",
    "amount": -500,
    "description": "ATM 提款",
    "balance": 50000.00
  }
]
```

### 測試資料

| 帳號 | 戶名 | 餘額 (TWD) |
|------|------|-----------|
| ACC001 | 張先生 | 50,000.00 |
| ACC002 | 李小姐 | 120,000.00 |
| ACC003 | 王先生 | 8,500.50 |

## 🔧 技術架構詳解

### .NET Core 版本架構

#### 架構模式
- **模式：** MVC (Model-View-Controller)
- **分層：** Controller → Service → Data (模擬)
- **依賴注入：** ASP.NET Core 內建 DI Container

#### 關鍵元件

##### 1. Program.cs - 應用程式進入點
**職責：**
- 配置 Web 應用程式建構器
- 註冊服務到 DI 容器
- 設定中介軟體管線（CORS、路由）
- 啟動應用程式

**關鍵程式碼：**
```csharp
var builder = WebApplication.CreateBuilder(args);

// 註冊服務
builder.Services.AddControllers();
builder.Services.AddSingleton<AccountService>();  // 單例模式

var app = builder.Build();

// CORS 設定（允許跨域請求）
app.UseCors(policy => policy
    .AllowAnyOrigin()
    .AllowAnyMethod()
    .AllowAnyHeader());

app.MapControllers();
app.Run();
```

##### 2. AccountController.cs - API 控制器
**職責：**
- 定義 RESTful API 端點
- 處理 HTTP 請求和回應
- 驗證輸入參數
- 錯誤處理（404 Not Found）

**設計模式：**
- 建構子注入（Constructor Injection）
- RESTful 路由設計

**關鍵註解：**
- `[ApiController]` - 標記為 API 控制器
- `[Route("api/account")]` - 基礎路由
- `[HttpGet("{accountId}/balance")]` - GET 端點定義

**程式碼結構：**
```csharp
[ApiController]
[Route("api/account")]
public class AccountController : ControllerBase
{
    private readonly AccountService _service;
    
    // 建構子注入
    public AccountController(AccountService service)
    {
        _service = service;
    }
    
    // 端點 1：查詢餘額
    [HttpGet("{accountId}/balance")]
    public IActionResult GetBalance(string accountId)
    {
        // 業務邏輯委派給 Service
        // 錯誤處理：帳戶不存在回傳 404
    }
    
    // 端點 2：查詢交易
    [HttpGet("{accountId}/transactions")]
    public IActionResult GetTransactions(string accountId)
    {
        // 業務邏輯委派給 Service
        // 錯誤處理：帳戶不存在回傳 404
    }
}
```

##### 3. AccountService.cs - 業務邏輯服務
**職責：**
- 實作核心業務邏輯
- 管理帳戶資料（模擬資料庫）
- 產生交易記錄

**資料結構：**
- 使用 `Dictionary<string, decimal>` 模擬帳戶資料庫
- 使用 `List<Transaction>` 儲存交易記錄

**關鍵方法：**
```csharp
public class AccountService
{
    // 靜態資料（模擬資料庫）
    private static readonly Dictionary<string, decimal> _accounts = new()
    {
        { "ACC001", 50000.00m },
        { "ACC002", 120000.00m },
        { "ACC003", 8500.50m }
    };
    
    // 查詢餘額
    public decimal GetBalance(string accountId)
    
    // 檢查帳戶是否存在
    public bool AccountExists(string accountId)
    
    // 取得近期交易（模擬 3 筆）
    public List<Transaction> GetRecentTransactions(string accountId)
}

// 交易資料模型
public class Transaction
{
    public DateTime Date { get; set; }
    public decimal Amount { get; set; }
    public string Description { get; set; }
    public decimal Balance { get; set; }
}
```

## 🔄 遷移對照表

### 框架對照

| 層面 | .NET Core | Java/Spring Boot |
|------|-----------|------------------|
| **Web 框架** | ASP.NET Core | Spring Boot |
| **控制器** | `ControllerBase` | `@RestController` |
| **依賴注入** | 內建 DI | Spring IoC Container |
| **配置檔** | `.csproj` | `pom.xml` / `build.gradle` |

### 型別轉換

| .NET Core | Java | 說明 |
|-----------|------|------|
| `decimal` | `BigDecimal` | 金額精度（重要！） |
| `DateTime` | `LocalDateTime` | 日期時間 |
| `Dictionary<K,V>` | `Map<K,V>` | 鍵值對集合 |
| `List<T>` | `List<T>` | 列表 |
| `string` | `String` | 字串 |
| `IActionResult` | `ResponseEntity<?>` | HTTP 回應 |

### 註解轉換

| .NET Core | Java/Spring Boot | 用途 |
|-----------|------------------|------|
| `[ApiController]` | `@RestController` | 標記 REST 控制器 |
| `[Route("path")]` | `@RequestMapping("/path")` | 基礎路由 |
| `[HttpGet("path")]` | `@GetMapping("/path")` | GET 端點 |
| `[HttpPost]` | `@PostMapping` | POST 端點 |
| 建構子注入 | `@Autowired` / 建構子 | 依賴注入 |
| `AddSingleton<T>` | `@Service` | 服務註冊 |

### 回應處理

| .NET Core | Java/Spring Boot |
|-----------|------------------|
| `Ok(data)` | `ResponseEntity.ok(data)` |
| `NotFound(data)` | `ResponseEntity.notFound().build()` |
| `BadRequest()` | `ResponseEntity.badRequest()` |
| `StatusCode(code)` | `ResponseEntity.status(code)` |

## 📊 架構圖

### 請求流程

```
客戶端請求
    ↓
[HTTP GET /api/account/{id}/balance]
    ↓
AccountController
    ├─ 接收請求
    ├─ 驗證參數
    └─ 呼叫 Service
        ↓
    AccountService
        ├─ 查詢資料（Dictionary）
        ├─ 業務邏輯處理
        └─ 回傳結果
            ↓
    AccountController
        ├─ 格式化回應
        └─ 回傳 JSON
            ↓
        客戶端接收
```

## 🎓 遷移重點

### 必須保持一致的項目

1. ✅ **API 端點路徑** - 完全相同
2. ✅ **回應格式** - JSON 結構一致
3. ✅ **測試資料** - 三個帳戶資料相同
4. ✅ **業務邏輯** - 計算邏輯一致
5. ✅ **錯誤處理** - 404 行為一致

### 需要特別注意的項目

1. 🔴 **金額精度** - `decimal` → `BigDecimal`（避免浮點數誤差）
2. 🟡 **日期時間** - `DateTime` → `LocalDateTime`（時區處理）
3. 🟢 **依賴注入** - 框架機制不同但概念相同
4. 🟢 **CORS 設定** - Spring Boot 需要額外配置

## 📝 補充說明

### 為什麼選擇這個架構？

1. **簡單清晰** - 適合教學和示範
2. **分層明確** - Controller-Service 分離
3. **易於遷移** - 結構對應清楚
4. **無外部依賴** - 使用記憶體資料，降低複雜度

### 實際專案的差異

在真實的銀行系統中，還會包含：
- 資料庫層（Entity Framework / JPA）
- 認證授權（JWT / OAuth）
- 日誌記錄（Logging）
- 例外處理中介軟體
- 單元測試和整合測試
- API 文檔（Swagger / OpenAPI）
